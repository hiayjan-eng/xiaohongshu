# 收藏复活系统 UX 规格

## 1. 当前 UX 状态

当前 Web 已经形成完整页面集合：Welcome、今日复活、导入中心、旧收藏扫描 Beta、智能专辑、收藏池、搜索、详情、设置、QA 和真实试用。视觉系统已有 5 套主题变量、奶油浅色背景、柔和卡片和响应式样式。

主要问题是页面增长来自多轮补丁，信息层级还没有完全统一。`apps/web/src/App.tsx` 中 Dashboard、Import、OldImport、Search、Pool、Detail、SmartAlbums、Settings 都在同一文件内，交互语义容易彼此漂移。后续 UX 规格的作用是固定页面结构和文案语义，避免实现继续靠临时补丁堆上去。

## 2. Web 桌面端导航

目标导航：

1. 今日复活
2. 导入中心
3. 智能专辑
4. 搜索
5. 收藏池
6. 旧收藏 Beta
7. 设置

QA 和真实试用不进入普通侧边栏。它们保留路由，但入口在“设置 → 开发与测试”内，默认折叠，只在 development、`?dev=1` 或 `localStorage developerMode=true` 时显示。

导航视觉规则：

- 左侧宽度约 236-248px，保持 App 感而非后台菜单。
- 当前页使用柔和主题底色和主色文字。
- 旧收藏 Beta 要标注 Beta，不要暗示普通用户无需安装扩展也能直接扫描。
- 计划库不再作为主流程导航。PlanCard 是今日复活里的轻量计划，不是单独重型计划系统。

## 3. 手机端导航

未来原生 App 底部 Tab：

- 今日
- 搜索
- 收藏
- 专辑
- 我的

手机端不承担旧收藏批量扫描。手机首要任务是接收系统分享、快速搜索、查看今日行动和复活一条收藏。旧收藏扫描入口只可作为说明页链接到桌面扩展方案，不提供“读取完整收藏夹”的承诺。

## 4. 今日复活页面

当前状态：Dashboard 已有全局搜索、今日计划卡、收藏搜索和数据摘要，但真实反馈是桌面布局还会显得拥挤，搜索结果可能需要向下滚动。

目标布局：

```text
┌────────────────────────────────────────────┐
│ 顶部全局搜索：找回任何收藏                  │
├────────────────────────────────────────────┤
│ 今天，从一条收藏开始                       │
│ 副文案：搜一个地点、技能、菜名、工具或主题   │
├───────────────────────┬────────────────────┤
│ 找一条收藏，今天复活  │ 今日轻量统计         │
│ [搜索框             ] │ 已索引 / 已复活 / 连续 │
│ 5 条搜索结果          │                    │
├───────────────────────┴────────────────────┤
│ 今日计划 PlanCard 列表                      │
├────────────────────────────────────────────┤
│ 最近导入 / 最近搜索 / 主题洞察              │
└────────────────────────────────────────────┘
```

交互规则：

- Dashboard 内部搜索必须在首屏可见，不应被其他卡片挤到下面。
- Enter 和按钮都触发搜索，最多展示 5 条结果。
- 每条结果显示标题、一级/二级分类、收藏用途、打开原帖、查看收藏、复活这条。
- “查看全部搜索结果”跳转 `/search?q=关键词`。
- 没有今日计划时显示“今天还没有安排。搜一条收藏，做最小的一步。”，不要只显示多个 0。

## 5. 收藏池布局

当前状态：收藏池已有卡片/表格、搜索、分类和状态筛选，并能显示“尚未复活”。

目标布局：

```text
收藏池
这里整理扫描和导入的收藏。需要时，再选择一条复活。

[搜索收藏] [主题筛选] [用途筛选] [状态筛选] [卡片/表格]

┌ 收藏卡片 ───────────────────────┐
│ displayTitle                     │
│ 内容主题 / 子主题 · 收藏用途      │
│ 摘要或信息不足提示               │
│ 关键词 chips                     │
│ 打开原帖 / 查看收藏 / 复活这条    │
└─────────────────────────────────┘
```

规则：

- 旧 `category=其他` 在展示层映射为“暂存 / 待补充备注”。
- 标题优先显示 `displayTitle`，缺失时显示“标题待补充”。
- 没有 URL 时不显示可点击“打开原帖”，显示“暂无原帖链接”。
- 批量操作要有预览和撤销，不做无提示覆盖。

## 6. 智能专辑布局

当前状态：`SmartAlbumsView` 已有候选、已确认、待确认新增、已归档标签，也有 `/albums/:albumId` 详情和确认弹窗。问题在于卡片密度和生命周期语义仍容易让用户混淆。

目标列表：

```text
智能专辑
系统按“内容主题”和“收藏用途”整理收藏，你确认后才会长期收纳。

[待确认 8] [已确认 2] [待确认新增 5] [已归档 1]

┌ 专辑候选 ───────────────────────┐
│ 主题专辑 / 用途专辑 · candidate  │
│ 专辑标题                         │
│ 一句话说明 whyThisAlbum          │
│ 收藏 12 条 · 推荐先看 3 条        │
│ keywords                         │
│ [查看并整理] [确认这个专辑] [更多] │
└─────────────────────────────────┘
```

按钮语义：

- 查看并整理：只进入详情，不改变状态。
- 确认这个专辑：打开确认弹窗，确认后变成 `confirmed`。
- 归档这个候选：从待确认隐藏，进入已归档，不删除收藏。
- 暂不需要：禁用该文案，统一改为“归档这个候选”。

详情页：

```text
专辑标题
类型 / 状态 / 收藏数量 / 自动收纳设置

为什么被放在一起
匹配依据
待确认新增
推荐收藏
全部收藏

批量操作：移动主题 / 添加用途专辑 / 移出当前专辑
```

## 7. 扫描与导入布局

导入中心目标：

- 旧收藏扫描是主入口，但必须说明需要桌面 Chrome / Edge 扩展 Beta。
- 新收藏手动导入是次入口，支持粘贴链接或整段分享文本。
- 批量链接、浏览器书签、手机分享入口如果未完成，只显示 Coming soon，不给用户可点击的假入口。

旧收藏扫描页目标：

```text
旧收藏扫描 Beta
需要先安装本地浏览器扩展 Beta。它不是 Chrome/Edge 商店正式扩展。

[下载旧收藏扫描 Beta ZIP]
[复制 chrome://extensions] [复制 edge://extensions]
[我已安装，检测扩展] [打开小红书收藏页]

扩展连接状态
扫描状态镜像：总数、新增、重复、缺标题、缺链接、待导入、阶段、最近更新时间
诊断信息：READY、PING、PONG、扩展版本、协议版本、失败原因
```

扫描 0 条或 0 新增必须区分：

- 页面不是小红书页面。
- 疑似不是收藏页。
- 扫描器未注入。
- 扩展已连接但扫描状态未同步。
- 本轮全部是重复。
- 当前页面未加载收藏卡片。
- 断点恢复后没有新增。
- 遇到登录提示、验证码或页面异常。

## 8. 搜索页面

当前状态：`/search` 已支持 `?q=` 恢复，搜索范围包括收藏字段、行动卡和智能专辑。

目标结构：

```text
找回你收藏过的那一条
[搜索框：大理 / Codex / 低卡晚餐 / 周末去处]

结果 12 条
┌ 结果卡 ────────────────────────┐
│ 标题                             │
│ 内容主题 / 子主题 · 收藏用途      │
│ 摘要                             │
│ 匹配原因 chips                   │
│ 打开原帖 / 查看收藏 / 复活这条    │
└─────────────────────────────────┘
```

规则：

- 空查询不跳转。
- 查询变化同步 URL，刷新保持查询和结果。
- 结果必须显示匹配原因。
- 打开原帖来自搜索时记录 SearchLog。

## 9. 收藏详情与行动卡

收藏详情是 SavedItem 详情，不等于 ActionCard。默认展示收藏索引：

- displayTitle
- 来源 URL 或“暂无原帖链接”
- 内容主题 / 子主题
- 收藏用途
- 判断原因
- 摘要
- keywords / entities
- 用户备注
- 分类纠正入口
- “复活这条”

ActionCard 只在用户选择复活后生成。ActionCard 必须包含：

- whySaved
- nextAction
- openOriginalFocus
- output
- doneCriteria
- avoidDoing
- ifInfoMissing
- followUp
- tasks

低信息量收藏要显示补充备注和重新生成入口。

## 10. PlanCard

PlanCard 是用户主动加入今日计划后的轻量计划卡，不是多日计划系统。

字段展示：

- 标题
- 来源收藏 `sourceTitle`
- 下一步行动 `oneNextStep`
- 预计时间 `estimatedMinutes`
- 计划日期 `plannedDate`
- 完成标准 `doneCriteria`
- 状态

操作：

- 开始
- 完成
- 延期到明天
- 更换日期
- 取消计划
- 查看来源收藏

取消和延期不能复制新卡；完成要更新复活值和今日统计。

## 11. 设置页

设置页包含：

- 外观主题。
- AI 状态：ServerAIProxy、provider、model、fallback、Key 是否配置。
- 同步状态：当前 localStorage / 未来 Supabase。
- 本地测试数据管理：导出、清空、重新整理旧收藏。
- 旧扫描文本修复：预览、导出备份、应用、撤销。
- 开发与测试：QA、真实试用，默认折叠。

清空数据必须二次确认，文案明确“只清空当前浏览器，不影响线上项目代码”。

## 12. 状态文案

空状态：

- 今日无计划：今天还没有安排。搜一条收藏，做最小的一步。
- 收藏池为空：还没有收藏索引。先扫描旧收藏，或手动导入一条真实收藏。
- 搜索无结果：还没找到，但这不代表它不存在。换个地点、技能、场景或工具名试试。
- 专辑为空：导入几条收藏后，系统会按主题和用途整理候选专辑。

错误状态：

- 导入失败：这段内容信息太少，建议补充标题或备注后再生成。
- 无原帖链接：暂无原帖链接，但仍可搜索和整理这条收藏。
- 扩展未连接：没有检测到扩展，请确认已加载解压后的 Beta 扩展并刷新页面。
- 扫描 0 新增：显示具体原因，不使用“没有结果”一笔带过。

加载状态：

- 导入：正在整理收藏索引...
- 复活：正在把这条收藏变成行动卡...
- 专辑：正在整理主题和用途...
- 文本迁移预览：正在检查旧扫描文本...

## 13. 确认、归档和撤销语义

- 确认：把候选变成长期保留的用户认可对象。
- 归档：从当前工作流隐藏，但不删除收藏、不删除索引、不删除分类。
- 删除：真正移除数据，必须二次确认，并优先提供导出备份。
- 撤销：恢复最近一次用户手动操作，不跨越多轮自动生成。
- 自动收纳：只对 confirmed 专辑生效，用户手动移出的收藏不得被自动塞回。

## 14. 响应式规则

- 桌面主内容最大宽度约 1240-1320px。
- Dashboard 桌面两列，低于 1180px 变一列。
- 智能专辑桌面最多两列，手机一列。
- 收藏池卡片最小宽度不低于 280px。
- 390px 手机宽度不允许横向溢出。
- 行动按钮在手机端堆叠，主按钮优先。

## 15. 设计 Token

当前已有 CSS variables：

- `--color-page`
- `--color-surface`
- `--color-surface-soft`
- `--color-primary`
- `--color-primary-hover`
- `--color-primary-dark`
- `--color-accent`
- `--color-accent-soft`
- `--color-text`
- `--color-muted`
- `--color-subtle`
- `--color-border`
- `--color-success`
- `--color-warning`

目标：所有主要卡片、按钮、标签、状态、边框和 toast 使用这些变量，不继续散落硬编码颜色。

## 16. 字体、间距、色彩和动效

- 字体：系统 sans-serif，中文 fallback 使用 PingFang SC / Microsoft YaHei。
- 页面标题：32/36，权重 700。
- 模块标题：18/22，权重 650。
- 正文：14/22。
- 卡片圆角：20-24px。
- 按钮圆角：999px 或 12px，按现有视觉统一。
- 动效：fade up、hover 上浮、focus 柔和描边。
- 支持 `prefers-reduced-motion`，减少动画时关闭撒花、漂浮和大部分 transition。
